# AI-oral-exam
# 🎙️ AI 口试系统开发计划书

## 📋 项目预期模块

### 1️⃣ 人机交互模块
> **核心目标**：打通语音闭环，实现无障碍的人机语音交互功能。

*   支持语音输入与识别（ASR）
*   支持大模型文本生成回复
*   支持语音合成与播报（TTS）

### 2️⃣ 数据整理模块
> **核心目标**：实现基于自定义材料的个性化测试与客观打分。

*   **规则解析**：支持上传考试规则文件，作为后续打分和控场的约束条件。
*   **资料解析**：支持上传同学的个性化资料（目前支持格式：**图片**、**文本报告**），并基于此生成针对性考题。

### 3️⃣ 多智能体协作模块
> **核心目标**：模拟真实考委会，通过多 LLM 协作完成“考”与“评”。

*   **面试官矩阵**：利用多个 LLM 组合扮演不同角色的考官（如主考官、专业追问官等），对同学进行交叉提问。
*   **评分委员会**：考试结束后，多智能体协同对同学的临场表现进行多维度打分与点评。

### 4️⃣ 记录模块
> **核心目标**：保障考试过程的鲁棒性与数据完整性。

*   **断点续考**：支持考试过程中的随时中断与无缝恢复。
*   **状态持久化**：实时且持续地保存特定同学在单次考试中的完整交互记录与状态。
